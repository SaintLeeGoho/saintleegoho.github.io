    # HTML基础

    # HTML前言

    • 两位先驱：艾伦 麦席森 图灵（人工智能之父）、约翰 冯 诺依曼（现代计算机之父）
    • 两种架构：C/S架构、B/S架构（Client、Browser、Server）
    • 前端范围：微信小程序、React Native、uni-app+Vue、node.js、ECHARTS
    • 浏览器排名：Chrome（Blink内核）、IE（Trident内核）、FireFox（Gecko内核）、Safari（webkit内核）、Opera（Presto内核改Blink内核）、UC、Android、Samsung Internet、Edge Legacy......
    • 网页组成：结构（HTML）、表现(CSS)、行为(JavaScript)

    # HTML简介

    全称：HyperText Markup Language，译为标记语言，超文本，相比普通文本内容更丰富

    文件后缀：.html

    编辑器：VisualStudioCode（安装简体中文语言包以及LiveServer插件实现实时查看页面效果）

    # HTML发展历史

    • 1993年06月：HTML由IETF(1985)作为草案发布
    • 1995年11月：HTML2.0由IETF正式发布
    • 1996年09月：IETF关闭HTML工作组，由W3C(1994)制订HRTML规范
    • 1997年01月：HTML3.2由W3C发布
    • 1997年12月：HTML4.0由W3C发布
    • 1999年12月：HTML4.0.1由W3C发布
    • 2000年左右：Adobe Flash Player发布
    • 2000年：XHTML1.0由W3C发布，必须小写
    • 2001年：XHTML1.1由W3C发布
    • 2002年：W3C继续制订XHTML规范
    • 2004年：WHATWG、W3C分道扬镳
    • 2014年：HTML5由W3C正式发布

    1. 用于给标签提供附加信息

    2. 可以写在起始标签或单标签中，形式为：<标签名 属性名="属性值" 属性名="属性值"...>

    3. 有些特殊的属性，没有属性名，只有属性值，例如：

      <input disabled>
      1

    4. 注意点：

      • 不同的标签有不同的属性；也有一些通用属性（在任何标签内都能写）

      • 属性名、属性值不能乱写，都是W3C规定好的

      • head标签的title标签可以指定网页的标题

    # HTML文档声明

    1. 作用：告诉浏览器当前网页的版本

    2. 写法：

      • 旧写法：要依网页所用的HTML版本而定，写法有很多

        <!DOCTYPE html>

    # HTML字符编码

    1. 计算机对数据的操作：

      • 存储时，对数据进行编码
      • 读取时，对数据进行解码

    2. 编码、解码会遵循一定的规定——字符集

    3. 常见字符集：

      • ASCII：美国设计，大写字母、小写字母、数字、一些符号，共计7位128个
      • ISO 8859-1：欧洲扩充，较于ASCII扩充了一些希腊字符等，共计8位256个
      • GB2312：国标，中国扩充，收录了6763个常用汉字，682个字符
      • GBK：GB2312的扩充，收录了的汉字和符号到达20000+，支持繁体中文
      • UTF-8：万国码，包含世界上所有语言的所有文字与符号

    4. 使用原则：

      • 存储时，务必使用合适的字符编码，否则无法存储，数据会丢失

      • 存储时采用哪种方式编码，读取时就采用哪种方式解码，否则数据错乱（乱码）

        例如下面文字中，包含有：中文、英文、泰文、日文

        我爱你

        I love you

        ผมรักคุณมาก

        愛しています

        若使用ISO8859-1编码存储，在存入的那一刻，就出问题了，因为ISO8859-1仅支持英文

        为保证所有的输入都能正常的存储和读取，现在几乎全部采用UTF-8编码。

    5. 为了让浏览器在渲染html文件时不犯错误，可以通过meta标签配合charset属性指定字符串

      <head>
	<meta charset="utf-8" />
</head>

    # HTML设置语言

    1. 主要作用：

      • 让浏览器显示对应的翻译提示
      • 有利于搜索引擎优化

    2. 具体写法：

      <html lang="zh-CN">
      1

    3. 拓展：lang的编写规则

      1. 第一种写法（语言-国家/地区），例如：
      • zh-CN：中文-中国大陆（简体中文）
      • zh-TW：中文-中国台湾（繁体中文）
      • zh：中文
      • en-US：英语-美国
      • en-GB：英语-英国
      1. 第二种写法（语言-具体种类）已不推荐使用，例如：
      • zh-Hans：中文-简体
      • zh-Hant：中文-繁体
      1. W3School上的说明：《语言代码参考手册》 (opens new window)、《国家/地区代码参考手册》 (opens new window)
      2. W3C官网上的说明：《Language tags in HTML》 (opens new window)

    # HTML标准结构

    1. HTML标准结构如下：
    ```html
<!DOCTYPE html>
<html lang="zh_CN">
<head>
    <meta charset="UTF-8">
    <!-- 移动端网页需要 -->
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <title>我是一个标题</title>
</head>
<body>
    
</body>
</html>
```

    # HTML排版标签

    标签名 标签含义 单/双标签
    h1~h6 标题 双
    p 段落 双
    div 没有任何含义，用于整体布局 双
    1. h1最好写一个，h2~h6能适当多写
    2. h1~h6不能互相嵌套，例如：h1标签中最好不要写h2标签
    3. p标签特殊。它里面不能有：h1~h6、p、div标签

    # HTML语义化标签

    1. 概念：用特定的标签，去表达特定的含义
    2. 原则：标签的默认效果不重要（后期可以通过CSS随便控制效果），语义最重要
    3. 举例：对于h1标签，，效果是文字很大（不重要），语义是网页主要内容（很重要）
    4. 优势：
      • 代码结构清晰可读性强
      • 有利于SEO（搜索引擎优化）
      • 方便设备解析（如屏幕阅读器、盲人阅读器等）

    # 块级元素与行内元素

    1. 块级元素：独占一行（排版标签都是块级元素）

    2. 行内标签：不独占一行

    3. 使用规则：

      1. 块级元素中能写行内元素和块级元素（块级元素中几乎什么都能写）
      2. 行内元素中能写行内元素，但不能写块级元素
      3. 一些特殊的规则：
      • h1~h6不能互相嵌套
      • p中不要写块级元素
      1. 备注：marquee元素设计的初衷：让文字有动画效果，但如今我们可以通过CSS来实现，而且还可以实现得更加炫酷，所以marquee标签已经过时

    # 常用标签

    标签名 标签含义 单/双标签
    br 换行 单
    hr 分割横线 单
    pre 按原文显示（一般用于在页面中嵌入大段代码） 双

    注意：

    • 不要用br来增加文本之间的间隔，应用CSS的margin属性
    • hr标签的语义使分隔，如果不想要语义，只是想要画一条水平线，应用CSS

    # HTML图片标签

    标签名 标签语义 常见属性 单/双标签
    img 图片 src：图片路径
    alt:图片描述
    width：图片宽度，单位是像素
    height：图片高度，单位是像素pm    		 是

    总结：

    1. 尽量不同时修改图片的宽和高，可能造成比例失调
    2. alt属性的作用：
    • 搜索引擎通过alt属性，得知图片的内容
    • 当图片无法显示的时候，有些浏览器会呈现alt属性的值
    • 盲人阅读器会朗读alt属性的值

    # 相对路径、绝对路径

    1. 相对路径：以当前位置作为参考点，去建立路径
    符号 含义 举例
    ./ 同级 src="./photo.jpg"
    / 下一级 src="./a/photo.jpg"
    ../ 上一级 src="../photo.jpg"

    注意：

    • 相对路径中的./可以省略不写
    • 相对路径依赖的是当前位置，后期若调整了文件位置，那么文件中的路径也要修改

    1. 绝对路径：以根节点作为参考，去建立路径

      1. 本地绝对路径：C:/a/b/c/photo.jpg（很少使用）
      2. 网络绝对路径：http://www.atguigu.com/images/index_new/logo.png

      注意：

      • 使用本地绝对路径，一旦更换设备，路径处理起来比较麻烦，所以很少使用
      • 使用网络绝对路径，确实方便，但要注意：若服务器开启了防盗链，会造成图片引入失效

    # 常见图片格式

    1. jpg格式：

      • 概述：拓展名.jpg或.jpeg，是一种有损的压缩格式（丢弃肉眼不容易察觉的细节）
      • 主要特点：支持的颜色丰富，占用空间较小，不支持透明背景，不支持动图
      • 使用场景：对图片细节没有极高要求的场景，例如：网站的产品宣传图等。

    2. png格式：

      • 概述：拓展名为.png，是一种无损的压缩格式，能更高质量地保存图片

      • 主要特点：支持的颜色丰富，占用空间略大，支持透明背景，不支持动图

      • 使用场景：需要透明背景，或想要更高质量图片，例如：logo图、重要配图等。

    3. bmp格式：

      • 概述：拓展名为.bmp，不进行压缩的一种格式，能最大程度上保存图片更多细节

      • 主要特点：支持的颜色丰富，保留细节更多，占用空间极大，不支持透明背景，不支持动图

      • 使用场景：对图片细节要求极高的场景，例如：大型游戏中的图片等（网页很少使用）。

    4. gif格式：

      • 概述：拓展名为.gif，仅支持256种颜色，色彩呈现不是很完整
      • 主要特点：支持的颜色比较少，支持简单透明背景、支持动态图
      • 使用场景：网页中的动态图片

    5. webp格式：

      • 概述：拓展名为.webp，这种格式是谷歌推出的专门用来表示网页中图片的一个格式
      • 主要特点：它具备其他图片格式的所有优点，而且文件还特别的小，致命缺点是兼容性不好(老版本浏览器)
      • 使用场景：网页中各种图片

    6. base64格式：

      • 本质：将图片使用base64编码，这样可以将图片转换成字符，通过字符的形式引入图片
      • 主要特点：直接作为img标签的src属性的值即可，并且不受文件位置影响
      • 使用场景：一般都是一些较小的或需要和网页一起加载的图片才会使用base64

    # 常用表单控件

    1. 文本输入框：

      <input type="text">
      1

      常用属性如下：

      • name属性：数据的名称
      • value属性：输入框的默认输入框
      • maxlength属性：输入框最大可输入长度

    2. 密码输入框：

      <input type="password">
      1

      常用属性如下：

      • name属性：数据的名称
      • value属性：输入框的默认输入框（一般不用，无意义）
      • maxlength属性：输入框最大可输入长度

    3. 单选框：

      <input type="radio" name="sex" value="female">女
<input type="radio" name="sex" value="male">男
      1
      2

      常用属性如下：

      • name属性：数据的名称
      • value属性：提交的数据值
      • checked属性：让该单选框按钮默认选中

    4. 复选框：

      <input type="checkbox" name="hobby" value="smoke">抽烟
<input type="checkbox" name="hobby" value="drink">喝酒
<input type="checkbox" name="hobby" value="perm">烫头
      1

      常用属性如下：

      • name属性：数据的名称
      • value属性：提交的数据值
      • maxlength属性：让该复选框默认选中

    5. 隐藏域：

      <input type="hidden" name="tag" value="100">
      1

      用户不可见的一个输入区域，作用是：提交表单时携带一些固定的数据

      • name属性：数据的名称
      • value属性：真正提交的数据值

      • input标签编写的按钮，使用value属性指定按钮文字

      <input type="reset" value="点我重置">

      • input标签编写的按钮，使用value属性指定按钮文字

      • 不能编写type属性，其他属性，与普通文本输入框一致

    # 字符实体

    有些时候，在HTML中不能直接书写一些特殊符号，如：

    • 多个连续的空格（在网页中编写的多个空格默认情况会自动被浏览器解析为一个空格）

    • 比如字母两侧的大于小于号（可能会被认为是标签并解析）

    如果我们需要在网页中书写这些特殊的符号，则需要使用html中的实体（转义字符）实体的语法：&实体的名字;，如：

    实体名称 显示结果 描述
    &nbsp `` 空格
    &gt > 大于号
    &lt < 小于号
    &amp & 与
    &copy © 版权
    &reg ® 注册商标
    &trade ™ 商标
    &times × 乘号
    &divide ÷ 除号
    &iquest ¿ 倒问号
    &yen ¥ 元

    也可以写对应的实体编号，如&#160等同于&nbsp;

    更多的字符实体，可参考：HTML 字符实体 (opens new window)、HTML ISO-8859-1 参考手册 (opens new window)

    属性名 含义
    id 给标签指定唯一标识，注意：id是不能重复的
    作用：可以让label标签与表单控件相关联；也可以与CSS、JavaScript配合使用
    注意：不能在以下HTML元素中使用：
    <head>、<html>、<meta>、<script>、<style>、<title>
    class 给标签指定类名，随后通过CSS就可以给标签设置样式
    style 给标签设置CSS样式
    dir 内容的方向，值：ltr、rtl
    注意：不能在以下HTML元素中使用：
    <head>、<html>、<meta>、<script>、<style>、<title>
    title 给标签设置一个文字提示，一般超链接和图片用得比较多
    lang 给标签指定语言
    注意：不能在以下HTML元素中使用：
    <head>、<html>、<meta>、<script>、<style>、<title>

    # meta元信息

    1. 配置字符编码

      <meta charset="UTF-8">
      1

    2. 针对IE浏览器的兼容性设置

      <meta http-equiv="X-UA-Compatible" content="IE=edge">
      1

    3. 针对移动端的设置

      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      1

    4. 配置网页的关键字

      <meta name="keywords" content="8-12个以英文逗号隔开的单词/词语">
      1

    5. 配置网页的描述信息

      <meta name="description" content="80字以内的一段话, 与网站内容有关">
      1

    6. 针对搜索引擎爬虫配置

      <meta name="robots" content="可选值">
      1
      值 描述
      index 允许搜索爬虫索引此页面
      noindex 要求搜索爬虫不索引此页面
      follow 允许搜索爬虫跟随此页面上的链接
      nofollow 要求搜索爬虫不跟随此页面上的链接
      all 与index、follow等价
      none 与noindex、nofollow等价
      noarchive 要求搜索引擎不缓存页面内容
      nocache noarchive的替代名称

      <meta name="author" content="Saint">
      1

      <meta name="generator" content="Visual Studio Code">
      1

    9. 配置定义网页版权信息

      <meta name="copyright" content="2020-2024@版权所有">
      1

    10. 配置网页自动刷新
